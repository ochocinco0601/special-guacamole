# Discipline Authoring Guide

## Discipline File Format

Each file in `references/` follows this structure:

```markdown
# [Work Type] — Professional Disciplines

## [Author/Framework Name]

**Source:** [Key publication, year]

### Core Principles
1. **[Principle name]** — [one-line description]
2. ...

### Application Checklist
- [ ] [Verifiable question about the work]
- [ ] ...

### Cross-References
- [Other discipline] — when [co-application condition]
```

## Adding New Disciplines

When a work session reveals missing professional perspectives:

1. **Name the discipline and its canonical author(s)**
2. **Verify it's an established body of knowledge** (published, peer-validated, professionally referenced)
3. **Create or update the relevant discipline file** in `references/`
4. **Update the work type mapping table** in SKILL.md if a new work type emerged

The catalog is a living document. Every "I should have thought of that" moment is a signal to add a discipline entry.

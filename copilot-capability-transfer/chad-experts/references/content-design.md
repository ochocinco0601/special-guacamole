# Content Design — Professional Disciplines

Expert perspectives for content strategy, content design, and content transfer between mediums.

---

## Sarah Richards — Content Design

**Source:** *Content Design* (2017); GOV.UK content design principles

### Core Principles
1. **Content design is not writing** — it's choosing the best format (text, video, calculator, infographic) to meet the user need
2. **Answer, not essay** — lead with the answer; no one reads an essay before getting to what they need
3. **Start from user needs** — evidence-based, using data to determine what/when/where/how
4. **Pair writing** — content designer + subject matter expert; SME gets specialist knowledge down, designer challenges jargon
5. **Content crits** — collaborative critique sessions; designer presents, gets feedback from people not close to the project

### Application Checklist
- [ ] Is text the right format, or would a table/diagram/calculator serve better?
- [ ] Does the reader get their answer without reading surrounding context?
- [ ] Is every piece of jargon justified or replaced with plain language?
- [ ] Has the content been tested against actual user needs (not assumed needs)?

## Kristina Halvorson — Content Strategy

**Source:** *Content Strategy for the Web* (2009, 2nd ed. 2012)

### Core Principles — The Content Strategy Quad
1. **Substance** — what messages content communicates, in what genres/media
2. **Structure** — content models, metadata, information architecture of content items
3. **Workflow** — tools, people, practices for creating and maintaining content
4. **Governance** — policies ensuring content adheres to organizational strategy

### Application Checklist
- [ ] Is there a clear owner for this content? (governance)
- [ ] What happens when this content needs updating? Who does it? (workflow)
- [ ] Is the content modeled structurally, or just free-form text? (structure)
- [ ] Does the content serve a defined communication goal? (substance)

## Erin Kissane — Seven Qualities of Good Content

**Source:** *The Elements of Content Strategy* (2011, A Book Apart)

### Core Principles
1. **Appropriate** — for business, for users, for context
2. **Useful** — serves a real need
3. **User-centered** — designed around user needs, not org structure
4. **Clear** — unambiguous language and structure
5. **Consistent** — tone, style, terminology are predictable
6. **Concise** — says what it needs to, no more
7. **Supported** — has an owner, a lifecycle, a maintenance plan

### Application Checklist
- [ ] Can you name the user need this content serves? (useful, user-centered)
- [ ] Is the tone consistent with surrounding content? (consistent)
- [ ] Could any section be cut without losing meaning? (concise)
- [ ] Who maintains this after publication? (supported)

## Gerry McGovern — Top Tasks

**Source:** *Top Tasks: A How-To Guide* (2018); *Transform* (2016)

### Core Principles
1. **Top tasks dominate** — ~10 tasks account for the vast majority of what users want to accomplish
2. **Tiny tasks bury top tasks** — organization-driven content clutter obscures what matters
3. **The task is the strategy** — content serves task completion, not the other way around
4. **Removing content improves findability** — cutting 80-90% of content can increase success rates

### Application Checklist
- [ ] What are the top 3-5 tasks a user comes to this content to accomplish?
- [ ] Is any content present because the organization wants it there (not because users need it)?
- [ ] Could removing content improve the user's ability to find what matters?

## Carrie Hane & Mike Atherton — Structured Content

**Source:** *Designing Connected Content* (2018)

### Core Principles
1. **Content modeling** — research and model your subject area based on shared domain concepts
2. **COPE** — Create Once, Publish Everywhere (originated by NPR's Daniel Jacobson)
3. **Interface-independent** — content planned, developed, and stored outside any interface so any interface can use it
4. **Domain-driven** — content types identified as part of a domain model, not a page layout

### Application Checklist
- [ ] Is this content tied to a specific presentation, or structured for reuse?
- [ ] Could this content be consumed by a different medium without rewriting?
- [ ] Is the content model documented (types, relationships, metadata)?

## Plain Language Movement

**Source:** Orwell's "Politics and the English Language" (1946); Plain Writing Act of 2010; ISO 24495-1:2023

### Core Principles — Orwell's Six Rules
1. Never use a metaphor, simile, or figure of speech you're used to seeing in print
2. Never use a long word where a short one will do
3. If it is possible to cut a word out, always cut it out
4. Never use the passive where you can use the active
5. Never use jargon if you can think of an everyday English equivalent
6. Break any of these rules sooner than say anything outright barbarous

### Application Checklist
- [ ] Could any long words be replaced with short ones?
- [ ] Are there sentences that could lose words without losing meaning?
- [ ] Is passive voice used only when the actor is genuinely irrelevant?
- [ ] Would a non-specialist understand every term?

## Torrey Podmajersky — UX Writing

**Source:** *Strategic Writing for UX* (2019, 2nd ed. 2024)

### Core Principles
1. **Content-first design** — what would you say to a person? Start there.
2. **UX content patterns** — different interface contexts require different voice treatments
3. **Conversation-rooted** — UI text should feel like a helpful conversation, not a manual

### Application Checklist
- [ ] Do buttons, labels, and messages sound like something a helpful person would say?
- [ ] Are error messages actionable (tell the user what to do, not just what went wrong)?
- [ ] Is the voice consistent across the flow?

## Cross-Reference: Problem → Framework

| Content Problem | Primary Framework(s) |
|----------------|---------------------|
| What format should this content take? | Richards (content design — text may not be the answer) |
| Who owns and maintains this content? | Halvorson (governance + workflow quadrants), Kissane (supported) |
| Content is bloated / users can't find what matters | McGovern (top tasks — cut 80-90% to improve findability) |
| Content locked to one presentation | Hane/Atherton (structured content, COPE, interface-independent) |
| Writing is jargon-heavy or unclear | Plain Language (Orwell's six rules), Richards (pair writing) |
| UI text feels robotic or confusing | Podmajersky (conversation-rooted, content-first) |
| Content exists but lacks strategy | Halvorson (content strategy quad: substance, structure, workflow, governance) |

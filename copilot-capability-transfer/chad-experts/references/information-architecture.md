# Information Architecture — Professional Disciplines

Expert perspectives for organizing, structuring, and making information findable.

---

## Morville, Rosenfeld & Arango — Four Systems of IA

**Source:** *Information Architecture for the Web and Beyond* (1st ed. 1998, 4th ed. 2015)

### Core Principles — Four Systems
1. **Organization systems** — schemes (alphabetical, chronological, topical) and structures (hierarchy, database, hypertext) for dividing content into groups
2. **Labeling systems** — what you call things; terminology that provides context and meaning
3. **Navigation systems** — menus and links enabling exploration and movement
4. **Search systems** — how users enter criteria to find content

### The Three Circles
IA sits at the intersection of **Users** (needs, behaviors, experience), **Content** (volume, format, structure, ownership), and **Context** (business goals, culture, technology, constraints).

### Application Checklist
- [ ] Which of the four systems is failing? (diagnostic: if IA is broken, which system?)
- [ ] Are labels consistent and meaningful to the user (not just the organization)?
- [ ] Can a user navigate to their goal without using search?
- [ ] Does the organization scheme match how users think about the content (not how the org structures it)?

## Richard Saul Wurman — LATCH

**Source:** *Information Anxiety* (1989); coined "information architect" in 1976

### Core Principle — Five Ways to Organize Information
1. **L**ocation — by physical/virtual place
2. **A**lphabet — alphabetical listing
3. **T**ime — chronological order
4. **C**ategory — grouping by similarity/type
5. **H**ierarchy — by magnitude, importance, or rank

**Constraint check:** If your organizing scheme isn't one of these five, it's probably confused.

### Application Checklist
- [ ] Which LATCH scheme is this content organized by? Can you name it?
- [ ] Is the scheme consistent throughout, or does it switch mid-structure?
- [ ] Is this the right scheme for how users will look for this content?

## Dan Klyn — Ontology, Taxonomy, Choreography

**Source:** The Understanding Group (2009)

### Core Principles — Three Pillars of IA
1. **Ontology** — establishing particular meanings (vocabulary, definitions, what things imply)
2. **Taxonomy** — arrangement of parts to accomplish goals within/across contexts (categorization, hierarchy)
3. **Choreography** — the interactions between ontologies and taxonomies that create the navigation experience (invisible when done well, frustrating when done badly)

### Application Checklist
- [ ] Are terms defined consistently? Do the same words mean the same things throughout? (ontology)
- [ ] Are groupings logical and goal-oriented? (taxonomy)
- [ ] Does moving between sections feel natural and predictable? (choreography)
- [ ] If the IA feels wrong, which pillar is broken: meaning, grouping, or movement?

## Abby Covert — Making Sense of Messes

**Source:** *How to Make Sense of Any Mess* (2014, free online)

### Core Principles
1. **A "mess"** = a situation where interactions between people and information are confusing
2. **IA** = the way we arrange the parts of something to make it understandable as a whole
3. **IA is a universal skill** — not limited to websites; applicable to any organizational challenge
4. **Identify the mess before solving it** — name the confusion explicitly

### Application Checklist
- [ ] Can you name what's confusing about the current state? (identify the mess)
- [ ] Are you designing for the organization's mental model or the user's? (they differ)
- [ ] Would a non-specialist understand why things are organized this way?

## Resmini & Rosati — Pervasive IA (Cross-Channel)

**Source:** *Pervasive Information Architecture* (2011)

### Core Principles — Five Heuristics
1. **Place-making** — creating a sense of place in information environments
2. **Consistency** — maintaining logical and familiar patterns across touchpoints
3. **Resilience** — adapting to different users, needs, and seeking strategies
4. **Reduction** — managing information overload; simplifying choices
5. **Correlation** — connecting related information across contexts

### Application Checklist
- [ ] Does the user know "where they are" at all times? (place-making)
- [ ] Are patterns consistent across different pages/sections/channels? (consistency)
- [ ] Can different users with different goals navigate successfully? (resilience)
- [ ] Is information overload managed through progressive disclosure? (reduction)
- [ ] Are related items connected across sections? (correlation)

## Donna Spencer — Practical IA Methods

**Source:** *A Practical Guide to Information Architecture* (2010)

### Core Contribution
The practitioner's handbook — how to actually *do* IA work:
- User research methods for IA
- Content analysis and inventory
- Card sorting methodology
- Designing organization schemes
- Navigation design patterns

### Application Checklist
- [ ] Has the content been inventoried before organizing? (don't organize what you don't understand)
- [ ] Have users been involved in categorization (card sort, tree test)?
- [ ] Does navigation follow established patterns for this content type?

## Lisa Maria Marquis — Content-Driven IA

**Source:** *Everyday Information Architecture* (2019, A Book Apart)

### Core Principles
1. **Organizing changes perception** — how you structure information alters how it's perceived
2. **Content-first** — structural decisions follow from content analysis, not template assumptions
3. **Taxonomies serve users** — classification schemes exist to help people find things, not to satisfy organizational charts

### Application Checklist
- [ ] Were structural decisions made from analyzing the content, or from a template?
- [ ] Does the taxonomy reflect user mental models or organizational structure?
